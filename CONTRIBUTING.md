# Contributing to Finicify Frontend Test

Thank you for your interest in this project! This is a test submission project, but improvements and suggestions are welcome.

## Development Setup

1. Fork the repository
2. Clone your fork
3. Install dependencies: `npm install`
4. Start dev server: `npm run dev`
5. Make your changes
6. Test thoroughly
7. Submit a pull request

## Code Style

- Use TypeScript for all new code
- Follow existing component patterns
- Use Tailwind CSS for styling
- Keep components modular and reusable
- Add proper type annotations

## Component Guidelines

- Create reusable components in `/src/components`
- Page-level components go in `/src/pages`
- Use TypeScript interfaces for props
- Add JSDoc comments for complex functions
- Keep components focused and single-purpose

## Commit Messages

Use clear, descriptive commit messages:
- `feat: Add new feature`
- `fix: Fix bug in component`
- `docs: Update documentation`
- `style: Format code`
- `refactor: Refactor component`
- `test: Add tests`

## Pull Request Process

1. Update documentation if needed
2. Ensure all TypeScript compiles
3. Test on multiple screen sizes
4. Verify no console errors
5. Describe your changes clearly

## Questions?

Open an issue for any questions or suggestions!
